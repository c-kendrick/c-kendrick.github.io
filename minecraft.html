SkyFactions2

State Machine Spatial Math Custom YAML I/O RBAC

APIs Used: Vault, WorldEdit, WorldGuard

  • Spatial Logic & Protection: Programmed chunk-based coordinate maths to calculate territory claims. Leveraged the WorldEdit API to programmatically paste starting schematics and the WorldGuard API to generate protected 3D regions.
  • State Management & Data: Built an event-driven state machine to govern server-wide diplomacy (wars, alliances) and intercept complex entity damage events (projectiles, splash potions). Wrote custom YAML File I/O classes to persist faction states across server reloads.

Role-Based Access Control:

  • Hierarchical Tiers: Programmed a strict three-tier hierarchy (Leader, Officer, Member) natively saved in local data structures.
  • Command Gateway: Wrote rigorous validation checks for all routed commands, restricting critical geopolitical actions (declaring war, disbanding) to Leaders, while allowing Officers to manage day-to-day territorial operations.

WallStreetV2

Procedural Gen Event-Driven Spatial Math RBAC

APIs Used: WorldEdit

  • Procedural Generation: Hooked into the WorldEdit API to physically spawn large-scale mine regions, utilising a "RandomPattern" algorithm to generate terrain based on exact weighted percentages (e.g., 50% stone, 10% diamond).
  • Batch Processing: Implemented server-side schedulers to automate daily tasks, such as resetting corporate mines every 24 hours.

Role-Based Access Control:

  • Role Tiers: Built a custom corporate ranking system (CEO, Manager, Trusted, Banished) to manage employee access levels.
  • Event Interception & Boundaries: Intercepted player movement and interaction events to dynamically track chunk entry/exit, updating UI elements in real-time while calculating if a broken block fell within specific 3D region bounds to enforce build permissions.
  • Explicit Blacklisting: Implemented a "Banished" state to actively block specific players from interacting with company properties, overriding standard baseline permissions.
